Docker: структура и основные команды

Docker – это наиболее популярный инструмент для контейнеризации. Он включает:

Docker Engine
основной компонент, который запускает и управляет контейнерами.
s
Docker Hub
репозиторий для хранения и обмена Docker-образами.
Docker CLI
командная строка для управления контейнерами.

Основные команды Docker:

  • docker build: сборка Docker-образа из Docker-файла
  • docker run: запуск контейнера из образа
  • docker ps: отображение списка запущенных контейнеров
  • docker stop: остановка запущенного контейнера

Kubernetes: оркестрация контейнеров

Kubernetes – это система для оркестрации контейнеров, разработанная Google.

Она автоматизирует развертывание, управление и масштабирование контейнерных приложений.

Kubernetes.io

Основные компоненты Kubernetes:

  • Кластеры: группы серверов, на которых запускаются контейнеры
  • Поды: минимальные единицы вычислительных ресурсов, содержащие один или несколько контейнеров
  • Сервисы: абстракции для доступа к группам подов
  • Деплойменты: объекты, управляющие обновлениями подов и обеспечивающие их масштабирование

Поддержка других инструментов: Podman, CRI-O и другие

Кроме Docker и Kubernetes, существуют другие инструменты:

Podman:
альтернатива Docker, обеспечивающая совместимость с OCI (Open Container Initiative) и улучшенную безопасность.
CRI-O:
инструмент для работы с контейнерами в Kubernetes, который предоставляет интерфейс для управления контейнерами через CRI (Container Runtime Interface).
